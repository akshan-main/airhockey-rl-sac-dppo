# AirHockey Demonstrations (from a SAC expert)

5,000 episodes of demonstration data for the 2D air hockey environment,
generated by a Soft Actor-Critic agent trained via self-play. Each
transition records the bottom paddle's observation and the action the
SAC expert took.

This is the dataset used for **Stage 2 (behavior cloning)** of the
[`akshan-main/airhockey-dppo`](https://huggingface.co/akshan-main/airhockey-dppo)
Diffusion Policy. Stage 1 was the SAC training itself. Stage 3 is the
DPPO fine-tune on top of the BC checkpoint.

## Schema

A single compressed `.npz` file containing four arrays:

| Key        | Shape       | Dtype   | Description                                                |
|------------|-------------|---------|------------------------------------------------------------|
| `obs`      | (N, 10)     | float32 | Per-step observation, 10 normalized values (see below)     |
| `act`      | (N, 2)      | float32 | Per-step action in [-1, 1] (paddle accel x, accel y)       |
| `episode`  | (N,)        | int32   | Episode index (0..4999)                                    |
| `timestep` | (N,)        | int32   | Step within the episode                                    |

### Observation layout

All values are normalized to [0, 1] or [-1, 1]:

```
[ puck_x / W,
  puck_y / H,
  puck_vx / max_v,
  puck_vy / max_v,
  bot_paddle_x / W,
  bot_paddle_y / H,
  bot_paddle_vx / max_v,
  bot_paddle_vy / max_v,
  top_paddle_x / W,
  top_paddle_y / H ]
```

The bot paddle is the one being imitated. The top paddle is the opponent
(a frozen copy of the same SAC expert).

## How to load

```python
import numpy as np
from huggingface_hub import hf_hub_download

path = hf_hub_download("akshan-main/airhockey-demos", "demos.npz", repo_type="dataset")
d = np.load(path)
obs, act = d["obs"], d["act"]
print(obs.shape, act.shape)
```

## Source

Generated by `python -m airhockey.collect --expert ckpt/sac_expert.pt --episodes 5000`
from [github.com/akshan-main/airhockey-rl](https://github.com/akshan-main/airhockey-rl).

The SAC expert was trained via self-play for 1M environment steps on
the same env (Stage 1 of the pipeline). `airhockey/collect.py` then
runs two copies of the trained SAC agent against each other for the
specified number of episodes and saves every step from the bottom
paddle's perspective.

## Intended use

- Behavior cloning pretraining for diffusion policies, action chunking
  transformers, or any other imitation learning method that consumes
  (observation, action) pairs.
- Offline RL benchmarking on a small fully-observed continuous-control
  environment.

## Limitations

- Generated by a single SAC agent, so the demonstrations reflect one
  particular policy's play style. A more diverse dataset would mix
  multiple checkpoints at different skill levels.
- Only the bottom paddle's perspective is recorded; the top paddle's
  trajectory is observable through the obs but not labeled as a separate
  expert trajectory.
- The SAC agent was trained for 1M steps, so it's competent but not
  superhuman. Dataset quality is bounded by expert skill.
